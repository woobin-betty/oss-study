# 🗂️ Git & GitHub 정리

## ✅ 1. Git이란?
- **버전 관리 시스템(VCS)**이다.
- 파일이나 코드의 **변경 이력**을 기록한다.
- 과거 상태로 **되돌릴 수 있다.**
- 코드 작업을 **안전하게 관리**할 수 있다.

### ✔️ Git을 쓰면 좋은 이유
- 실수해도 **이전 버전으로 복구 가능**
- 작업 기록(커밋)이 남아 흐름 파악 쉬움
- 여러 사람이 동시에 작업해도 충돌을 줄일 수 있음

---

## ✅ 2. GitHub란?
- Git 저장소를 **온라인(클라우드)**에 올려 보관하는 서비스이다.
- 쉽게 말하면 **Git 프로젝트를 인터넷에 저장하는 공간**이다.
### ✔️ GitHub의 장점
- 어디서든 저장소 접근 가능
- 다른 사람과 협업 가능 (Pull Request, Issue 등)
- 프로젝트 백업 및 공개 가능

---

## ✅ 3. Git과 GitHub의 관계

| 항목 | Git | GitHub |
|---|---|---|
| 역할 | 버전 관리 도구 | Git 저장소를 보관하는 서비스 |
| 위치 | 로컬(내 컴퓨터) | 온라인(인터넷) |
| 인터넷 필요 여부 | ❌ 필요 없음 | ✅ 필요함 |
| 핵심 기능 | Commit, Branch, Merge | Pull Request, Issue, 협업 |

👉 **정리**
- **Git = 버전 관리 프로그램**
- **GitHub = Git 저장소를 업로드하는 웹사이트**

---

## ✅ 4. Git의 핵심 흐름

### 🔹 Working Directory
- 실제로 파일을 수정하고 작업하는 공간

### 🔹 Staging Area
- 커밋할 파일을 임시로 모아두는 공간

### 🔹 Repository
- 커밋들이 저장되는 실제 저장 공간

---

## ✅ 5. 자주 사용하는 Git 명령어

| 명령어 | 설명 |
|--------|------|
| `git init` | 새 Git 저장소 생성 |
| `git status` | 현재 상태 확인 |
| `git add 파일명` | 파일을 Staging Area에 추가 |
| `git commit -m "메시지"` | 변경 내용을 저장(버전 생성) |
| `git log` | 커밋 이력 보기 |
| `git remote add origin URL` | GitHub 저장소 연결 |
| `git push` | GitHub에 업로드 |
| `git pull` | GitHub 내용 가져오기 |

---

## ✅ 6. GitHub 저장소(Remote Repository)란?
- GitHub에 존재하는 저장소
- 내 컴퓨터 저장소(Local Repository)를 **온라인에 백업**하는 역할

---

## ✅ 7. Git을 쓰는 이유 총정리
- 코드 이력 관리
- 실수 복구 가능
- 협업에 최적화
- 안전한 백업 가능

---

## ✅ 한 문장으로 정리

> **Git은 버전을 관리하는 도구이고, GitHub는 그 저장소를 인터넷에 저장하는 공간이다.**
